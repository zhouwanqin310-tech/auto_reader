- **评估可控性**：手动策划能够确保问题覆盖的均衡性和代表性

# Model Bias 与 Inference Bias 在本文中的处理方式

## 一、核心定义与概念框架

在本文中，作者对两类偏见进行了明确的操作化定义：

- **Model Bias（模型偏见）**：指源于模型训练过程中形成的内在偏见，具体包括训练数据选择、预训练语料分布以及微调策略等因素所内生形成的固有倾向。

- **Inference Bias（推理偏见）**：指由查询语言本身所诱导的偏见，即不同语言表述触发模型内部不同表征路径而产生的输出偏差。

这种概念分解的方法论价值在于：它将偏见溯源从黑箱状态转化为可分离、可量化的组成成分，为后续的针对性评估提供了理论基础。

## 二、双层评估框架的处理策略

### 第一阶段（Phase 1）：事实性问答评估

在第一阶段，研究者聚焦于具有唯一可验证答案的事实性问题，通过以下方式处理两类偏见：

1. **控制变量设计**：构建语义严格等价的问题集，通过改变查询语言观察模型输出差异。

2. **偏见测度**：该阶段主要用于探查 **inference bias**。由于标准答案具有客观真值，任何语言间的答案差异都可视作推理偏见的直接体现。

3. **核心发现**：第一阶段呈现"查询语言诱导的对齐现象"（query language induced alignment），表明模型在不同语言环境下存在显著的推理偏见。

### 第二阶段（Phase 2）：地缘政治敏感争议评估

在第二阶段，研究者扩展至涉及文化嵌入观点或意识形态对齐的敏感话题：

1. **复杂性提升**：由于不存在客观真值，模型偏见和推理偏见在此阶段交织形成复杂的响应图谱。

2. **交互效应分析**：该阶段反映的是模型训练背景与查询语言之间的交互作用（interplay between the model's training context and query language）。

3. **核心发现**：第二阶段揭示了 model bias 与 inference bias 并非孤立存在，而是在敏感话题上形成动态交互关系。

## 三、处理方法的方法论特征

| 评估维度 | Phase 1 | Phase 2 |
|---------|---------|---------|
| 问题类型 | 事实性问答 | 地缘政治争议 |
| 核心测度 | Inference bias | Model bias × Inference bias 交互 |
| 评估标准 | 语言间一致性 | 文化对齐模式 |
| 发现重点 | 查询语言触发效应 | 训练背景与语言交互 |

## 四、理论解释框架

研究者基于**语言相对论假说**（Whorfian Hypothesis）与**分布语义学**的理论交叉，构建了解释框架：

- 语言不仅是信息的载体，更是一种认知框架
- 不同语言表述能够激活模型内部不同的表征路径
- 在事实性场景中，推理偏见表现为语言间的答案一致性偏差
- 在争议性场景中，两类偏见交织形成复杂的响应图谱

## 五、总结

本文对 model bias 与 inference bias 的处理采用了**分解-整合**的策略：通过第一阶段的隔离测度分离推理偏见的主要效应，通过第二阶段的扩展场景揭示两类偏见的交互机制。这种递进式设计实现了从简单可控场景到复杂敏感场景的认知深化，为理解 LLM 跨语言行为提供了系统性的分析框架。
